Для чего нужен справочник "Внутренние документы"

Справочник Внутренние документы служит для учета официальных информационно-справочных или организационно-распорядительных документов.

Документы в списке могут быть распределены по папкам. Для удобства просмотра можно включать или отключать иерархический просмотр. Кроме того, с помощью полей отбора Период, Вид документа и Организация можно быстро найти нужный документ.

Как работать со списком внутренних документов

В списке внутренних документов можно переключить режим просмотра: списком, по папкам, по категориям:

  • Списком - отображается только список документов.
  • По папкам - отображается структура папок и список документов. В список попадают документы, хранящиеся в выбранной папке.
  • По категориям - отображаются категории в виде дерева и список документов. В список попадают документы, имеющие выбранные в дереве категории.

Если настройка программы Использовать категории включена, то в списке документов появляется колонка со значком . В колонке отмечены документы, имеющие категории. В списке документов добавляется команда Подобрать категории . Команда открывает форму подбора категорий для выбранных в списке документов.

Если настройка программы Использовать штрихкоды включена, то доступна группа команд Штрихкоды со значком . В данной группе доступны следующие команды:

  • Добавить внешний штрихкод - позволяет добавить произвольный штрихкод документу. В дальнейшем по этому штрихкоду можно найти документ.
  • Сохранить штрихкод - позволяет сохранить изображение штрихкода в виде графического файла *.JPEG на диск.

В колонке Задачи с помощью пиктограммы показывается наличие у документов сформированных задач.

Если настройка Использовать электронные подписи включена, то в списке документов появляется колонка  Подписан. Значком  помечаются документы, подписанные электронной подписью.  

Документ может быть отправлен по электронной почте. Для этого в списке документов или карточке документа следует выполнить команду Отправить по электронной почте. На экран выводится форма отправки для ввода данных письма (получатель, тема, описание, вложенные файлы).

Как создать внутренний документ

Создать внутренний документ можно несколькими способами:

  • с помощью команды Создать,
  • с помощью загрузки файлов,
  • перетаскиванием (Drag&Drop) файла (или нескольких файлов) из Проводника.

В зависимости от вида внутреннему документу автоматически присваивается регистрационный номер.

Основные данные документа располагаются на закладке Реквизиты. Для документа следует задать: 

  • папку внутренних документов, в которой будет размещен документ;
  • пользователя, подготовившего документ;
  • руководителя (пользователя), утвердившего документ;
  • корреспондента и контактное лицо, с которыми связан документ. Реквизит отображается, если в настройках вида документа установлен флажок Вести учет по корреспондентам.
  • срок исполнения - задается дата исполнения документа. Реквизит отображается, если в настройках вида документа установлен флажок Использовать срок исполнения;
  • гриф доступа - если установлена настройка Использовать грифы доступа, то внутреннему документу нужно обязательно присвоить гриф доступа, выбрав его из числа заранее настроенных администратором грифов. Гриф определяет права доступа пользователей к внутреннему документу;
  • вид документа - показывается, если настройка программы Использовать виды внутренних документов установлена;
  • состояние документа - показывается, если настройка программы Использовать состояние документов установлена;
  • номенклатура дел - показывается, если для указанного вида документа в настройках установлено Вести учет по номенклатуре дел;
  • организация - в случае ведения документооборота для нескольких организаций (показывается, если настройка программы Использовать учет по организациям установлена);
  • вопрос деятельности - показывается, если настройка программы "Использовать вопросы деятельности" установлена;
  • не действует - если установлен, то следует в поле справа от флажка указать внутренний документ, на основании которого выполнена отмена (флажок показывается, если для указанного вида документа в настройках установлено "Учитывать недействующие документы").

Как работать с карточкой внутреннего документа

В блоке Хранение следует заполнить значение поля состав описывающее количество экземпляров и листов документа. Состав заполняется через специализированную форму. Если документ помещен (списан) в дело, то указывается данные дела. При списании документа в дело подсчет общего числа листов выполняется по значениям полей Количество листов и Листов в приложениях в карточке документа.
Поле учета состава документа показываются, если настройка программы Использовать состав документов установлена.

Если настройка программы Использовать учет доступа к персональным данным включена, то в карточке внутреннего документа на закладке Дополнительно появляется список физических лиц, персональные данные которых хранит этот документ. Этот список физических лиц будет автоматически распространен на все файлы, присоединенные к данному документу и их версии. Также при этом будет осуществляться протоколирование доступа к полю Содержание внутреннего документа в журнал регистрации. Не рекомендуется использовать другие поля внутреннего документа (например, Наименование или Комментарий) для хранения персональных данных физических лиц.

Если документ был только что создан, на карточке документа появляется кнопка  Заполнить по шаблону . При нажатии на нее открывается форма выбора шаблонов документов данного типа, если для данного типа документов в системе есть шаблоны.

Если настройка программы Использовать автозаполнение шаблонов файлов включена, то в списке файлов внутреннего документа становится доступной команда Заполнить файл данными документа, при выполнении которой осуществляется попытка заполнить файл данными документа, основываясь на правилах автоматического заполнения шаблонов файлов.

Если настройка Использовать электронные подписи включена, то в карточке документа добавляются группа команд ЭП (содержит команду Подписать, с помощью которой выполняется удостоверение документа электронной подписью) и закладка ЭП, в которой редактируются и проверяются электронные подписи. Если документ подписан, то его редактирование невозможно. Для редактирования нужно сначала удалить все подписи. Текущий пользователь может удалять только свои подписи. Право удаления всех подписей принадлежит администратору. Подпись можно сохранить в файл (например, для пересылки по электронной почте), а также посмотреть сертификат подписи.

Как настроить срок действия документа

Для настройки срока действия документа необходимо в карточке изменения срока действия документа заполнить реквизиты:

  • Действует с - дата начала действия договора,
  • Действуют по - указывается дата окончания действия, если она известна заранее, 
  • Бессрочный - устанавливается, если дата окончания договора неизвестна,
  • Продление - порядок продления договора.

Карточку изменения срока действия документа можно открыть при помощи клавиши F4 в поле Срок действия.

Посмотреть историю изменения срока действия документа Вы можете одним из двух способов:

  • Способ 1 – команда Карточка документов подменю Печать.
  • Способ 2 – команда История в карточке изменения срока действия.

Обратите внимание:
Изменение возможно только для тех документов, в карточке вида которых установлен флажок Учитывать срок действия (при установке флажка Является договором устанавливается автоматически).

Права доступа к внутренним документам определяются папкой, в которой размещен документ, и настраиваются администратором программы при необходимости.

Как прикрепить один или несколько файлов

К каждому внутреннему документу можно прикрепить один или несколько файлов.

При перетаскивании файла в карточку внутреннего документа, к которому уже присоединен файл с таким же именем, программа предлагает сохранить версию имеющегося файла либо не дает провести операцию, если имеющийся файл новее перетаскиваемого.

Для присоединения к внутреннему документу второго файла с таким же именем воспользуйтесь одним из следующих способов:

  • переименуйте имеющийся файл и добавьте второй файл любым из возможных способов.
  • присоедините второй файл, используя кнопку Добавить.Как поставить документ на контроль.

Как поставить внутренний документ на контроль

Внутренний документ можно поставить на контроль:

  • следить за сроком и ходом выполнения работ по документу,
  • переносить контрольный срок,
  • контролировать отдельных пользователей,
  • снимать документ с контроля,
  • выводить аналитические отчеты,
  • смотреть, какие документы находятся на контроле у ваших коллег.

Программа сама отследит истекающие и приближающиеся контрольные сроки и сообщит вам об этом по почте.  

Чтобы поставить документ на контроль, нажмите  в командной панели карточки или списка документов. При этом откроется Контрольная карточка документа, в которой нужно установить срок, указать контролируемое действие и исполнителя. 

Если документ уже контролировался ранее, кнопка  откроет список всех контрольных карточек когда-либо созданных по документу. В списке можно посмотреть, кто сейчас контролирует документ, открыть контрольные карточки и, при наличии прав, изменить параметры контроля.

Состояние контролируемого документа можно посмотреть в списках (колонка Контроль) и карточке (строка состояний):

    - документ на контроле и его контрольный срок не истек;
    - документ на контроле, но его контрольный срок истек;
    - документ снят с контроля.
    Значки с двумя означают, что документ находится на контроле у нескольких пользователей. Цвет по аналогии отражает состояние контроля у каждого из них.

                                      После постановки на контроль рекомендуется:

                                    • Настроить уведомления об истекающих или пропущенных сроках (в настройках программы),
                                    • Следить за всеми контролируемыми документами в окне На контроле.

                                        Как настроить рабочую группу

                                        Доступ к документу можно ограничить указанным списком лиц на закладке Рабочая группа.

                                        Рабочая группа – это список пользователей и ролей, которые могут просматривать этот документ. Заполнить рабочую группу может любой пользователь, у которого есть права на просмотр и изменение документов. Если рабочая группа не заполнена, то документ могут просматривать все пользователи.

                                        Обратите внимание:
                                        При заполнении обязательно внесите текущего пользователя, иначе программа не даст записать документ.

                                        Если для вида документа указано Автоматически вести состав рабочей группы, то в рабочую группу автоматически добавляются:

                                        • авторы процессов по документу,
                                        • исполнители процессов по документу,
                                        • пользователи, указанные в самом документе.

                                        Для отключения автоматического добавления участников в рабочую группу конкретного документа установите флажок Запретить автоматическое добавление участников рабочей группы на закладке Рабочая группа.

                                        Другие настройки внутренних документов

                                        На отдельной закладке расположены списки связанных документов.

                                        Если персональная настройка программы Использовать обзор документов установлена, то при открытии документа отображается закладка Обзор. На ней на которой расположена вся основная информация о документе, приложенные файлы, комментарий к документу и строка состояний.

                                        Для передачи внутренних документов другим исполнителям рекомендуется использовать процессы, доступные через меню Создать на основании, например - согласование, утверждение, исполнение, ознакомление и другие.

                                        Документ можно скопировать. Для этого следует выбрать документ-образец и выполнить команду Все действия - Скопировать. Создается новый документ, основные поля которого будут заполнены данными из документа-образца. Следует внести недостающие данные и исправить (при необходимости) поля, заполненные из документа-образца. 

                                        При копировании в создаваемый документ будут также скопированы файлы из документа-образца. Просмотреть и отредактировать файлы можно только после записи нового документа.

                                        При необходимости карточку документа можно распечатать.

                                        При необходимости можно напечатать регистрационный штамп документа. Если настройка программы Использовать штрихкоды включена, в регистрационный штамп будет добавлено изображение штрихкода документа.

                                        С помощью команд панели навигации можно открыть журнал передачи, просмотреть изменение состояния, историю переписки по данному документу.

                                        На закладке Визы находится список виз согласования данного документа. Вкладка доступна в карточке документа, если в настройках программы установлен флажок Использовать визы согласования. Визы согласования представляют собой записи о фактах и результатах согласования документов они могут создаваться вручную, либо автоматически из процесса Согласование. Создать новую визу или изменить существующую можно только, если документ находится в статусе На согласовании. При помощи команды Лист согласования можно распечатать результат согласования. При создании документа по шаблону список согласующих лиц шаблона копируется в список виз документа.

                                        Если включена настройка программы Использовать автоматическую категоризацию, на закладке Категории размещается статус автоматической категоризации. Если данный документ был обработан хотя бы одним правилом автоматической категоризации, будет установлена отметка Автоматическая категоризация этого документа выполнена. Если включена настройка программы Использовать проверку категорий, на закладке Категории отображается факт проверки правильности присвоения категорий данному документу.
