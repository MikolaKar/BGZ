Этот процесс обходит всех участников согласования (рецензентов) по заранее составленному списку, после чего возвращается к автору (инициатору процесса). В случае отрицательного результата согласования автор имеет возможность повторить согласование. В случае положительного результата автор может направить документ на утверждение при помощи меню "Создать на основании".

Общие сведения

При заполнении списка рецензентов можно указывать как непосредственно исполнителя (пользователя), так и роли и/или группы пользователей.

Предусмотрено три варианта обхода согласующих - "все вместе", "по очереди" и "смешанно". Если установлен тип согласования "все вместе", то задачи всем рецензентам формируются одновременно. При этом процесс возвращается к автору после завершения задач всеми согласующими.  Если установлен тип согласования "по очереди", то задача следующему рецензенту формируется только после завершения задачи предыдущему. При этом процесс сразу возвращается к автору, как только один из согласующих принял отрицательное решение. Если установлен тип согласования "смешанно", то для каждого исполнителя можно указать порядок формирования задачи - вместе с предыдущим либо после предыдущего. В результате тип согласования "смешанно" позволяет сочетать этапы одновременного и последовательного согласования.

Каждому из исполнителей на выполнение задачи отводится срок, указанный в карточке процесса.  При указании срока следует учитывать, что для варианта согласования "все вместе" ожидаемый срок завершения процесса совпадает со сроком, указанным в карточке. Тогда как для варианта согласования "по очереди" ожидаемый срок завершения процесса равен произведению количества исполнителей и срока, отведенного каждому исполнителю.

Этот процесс можно создавать непосредственно из карточки, например, исходящего документа, при помощи меню "Создать на основании".

Чем отличаются предметы процесса

Предметы процесса могут быть основными, вспомогательными и заполняемыми.

 Основной предмет содержит важную информацию о задаче и выступает в качестве ее основания.

 Вспомогательный предмет, как правило, содержит дополнительные данные, помогающие при выполнении задач. Состояние вспомогательного предмета не меняется по ходу выполнения задач процесса. В роли вспомогательного предмета может служить любой доступный объект в программе или файл с диска.

 Заполняемый предмет предусматривается для одной из задач процесса и вводится на этапе ее выполнения. Если для задачи предусмотрен заполняемый предмет, то ее нельзя выполнить, пока предмет не заполнен.

Чтобы добавить предмет в процесс , нажмите  Добавить предмет на командной панели. Если процесс создается на основании, то его основной предмет устанавливается автоматически. Для заполняемого предмета можно указать основание для заполнения, и задачу, в которой он должен быть введен.

Файлы в процесс добавляются также как и предметы. Файл можно добавить либо из списка файлов программы, либо с диска. При добавлении с диска можно добавить сразу несколько файлов.

Как добавить в процесс исполнителей

Чтобы добавить в процесс сразу несколько исполнителей или, например, целое подразделение:

  1. Нажмите кнопку Подобрать (она расположена над списком исполнителей).
  2. Выберите подразделение или группу пользователей. При этом в список исполнителей процесса попадут все сотрудники подразделения или группы. Для добавления нескольких сотрудников подразделения или группы используйте левую кнопку мыши с нажатой клавишей Ctrl.

Обратите внимание:
Добавить нескольких исполнителей можно только в не стартованный процесс.

Если процесс уже стартован, для подбора исполнителей следует использовать кнопку Добавить.

Как установить персональные сроки для каждого исполнителя

Для каждого исполнителя можно установить персональные сроки выполнения задач. Для этого необходимо нажать на кнопку Разные сроки в командной панели списка согласующих. После этого:

  • Если работа по задаче еще не начата – можно установить сроки в колонках Дней и Часов (например, при смешанном или последовательном согласовании);
  • Если работа по задаче уже ведется – сроки устанавливаются через команду Изменить срок.

Полезный совет:
При установке персональных сроков рекомендуется проверить расчетное время завершения задачи для каждого исполнителя (в поле Срок (датой)) и для всего процесса (в поле Ожидаемый срок согласования).

Если оба способа изменения сроков недоступны, значит:

  • Либо работа по задаче завершена, и изменить ее срок невозможно;
  • Либо не установлена настройка программы Вести учет переносов сроков задач.

Как остановить процесс

Для остановки процесса используйте команду Остановить. При этом остановленным будет помечен сам процесс, все его невыполненные задачи, а также все вложенные и подчиненные процессы и их невыполненные задачи.

Для того чтобы сделать остановленный процесс активным используйте команду Продолжить. При этом будут сделаны активными все вложенные и подчиненные процессы и их задачи. Остановленные задачи не отображаются в списках задач (например, Мои задачи).

Как прервать процесс

Нажмите Прервать на командной панели карточки процесса.

При этом с процессом происходят следующие изменения:

  • Процесс и его задачи прерываются,
  • Подчиненные процессы и их задачи прерываются,
  • Состояние согласования предмета очищается,
  • Визы, созданные в процессе, помечаются на удаление

Прервать процесс может его автор или администратор. Прерванный процесс, в отличие от остановленного нельзя возобновить.

Как установить главную задачу

С помощью поля Главная задача в карточке можно указать, что данный процесс создан в рамках вышестоящего процесса. Это поле заполняется автоматически при создании процессов на основании задач или документов, но может быть установлено вручную.

Дерево подчиненных процессов можно посмотреть при помощи команды Задачи в панели навигации карточки процесса. Поле Главная задача и просмотр дерева подчиненных процессов доступны только в том случае, если в программе разрешено использование подчиненных процессов (см. Настройка программы).

Как посмотреть состояние процесса

Для просмотра текущего состояния процесса (списка выполненных и не выполненных задач) используйте команду Задачи в панели навигации карточки процесса.

Как процесс влияет на предметы и задачи

Если предметом исполнения является исходящий или внутренний документ, то при старте этого процесса документу автоматически будет установлено состояние "На согласовании" и при успешном завершении процесса - состояние "Согласован".

Изменения, которые производятся в карточке начатого процесса (например, редактирование содержимого полей), наследуются незавершенными задачами по процессу.

При этом в карточке процесса доступны для редактирования только те поля, которые можно изменять после старта процесса.

Автор процесса имеет возможность изменять список согласующих после старта процесса. Доступно добавление, удаление, изменение согласующих и изменение их порядка. Для этого в карточке процесса необходимо изменить список согласующих и записать процесс. Можно изменять только тех исполнителей, которые не завершили свое согласование.

При добавлении нового исполнителя  в случае параллельного согласования автоматически будет создана задача для этого исполнителя. При удалении исполнителя, для которого уже сформирована задача, эта задача будет помечена на удаление. При изменении исполнителя,  для которого уже сформирована задача, новый исполнитель будет  записан в задачу.

Как настроить автоматический старт процесса

Процессы могут начинаться автоматически после определенного события.

Например, можно сделать так, чтобы при регистрации входящего документа вида коммерческое предложение, он автоматически направлялся на рассмотрение руководителю.

Как отложить старт процесса

Старт процесса можно отложить на любое удобное время.

Например, если Вы создали несрочный процесс для загруженного сотрудника, можно указать время старта процесса - через день. К тому времени Ваш коллега справится со срочными делами.
