Задачи являются средством коммуникации между пользователями, а также используются для автоматизации процессов.

Чтобы просмотреть список задач по процессу, выполните команду Все задачи на панели навигации.

С помощью команды Принять к исполнению можно отметить задачу как принятую. Принятая задача отображается обычным, а не жирным шрифтом в списке задач. Если задача имела ролевую адресацию (т.е. была назначена не конкретному сотруднику, а роли), то при выполнении команды Принять к исполнению адресация задачи будет изменена и она будет адресована лично сотруднику. Это удобно использовать для "захвата" задач в том случае, если исполнителями роли являются сразу несколько сотрудников. Отменить принятие к исполнению можно при помощи соответствующей команды в меню Все действия в карточке задачи.

Для завершения задачи служат кнопки в нижней части карточки. Варианты завершения задачи зависят от процесса, в рамках которого она выполняется.

Задача может быть перенаправлена другому исполнителю с использованием персональной или ролевой адресации. Для этого служит команда Перенаправить.

При персональной адресации исполнитель указывается явно из списка пользователей системы. А при ролевой адресации - необходимо указать роль и при необходимости, основной и дополнительный объекты адресации.

Если исполнитель не успевает выполнить задачу в срок, то можно отправить автору задачи запрос на перенос срока. Для этого служит команда Согласовать перенос срока в меню Все действия.

Если в задаче не определен предмет, то исполнитель, при необходимости, может установить его сам с помощью команды Добавить предмет. При этом предмет будет установлен для всех незавершенных задач по текущему процессу, а также в самом процессе.

Чтобы уточнить детали выполнения задачи, можно связаться с ее автором, выполнив команду Задать вопрос автору из меню Все действия.

Поле Результат регистрации отображает с каким результатом была завершена задача на регистрацию. Результат регистрации может принимать значения Не зарегистрировано или Зарегистрировано.

В случае если документ не был зарегистрирован и задача на регистрацию исполнена с отрицательным результатом, то автору процесса в карточке задачи ознакомления с результатом регистрации будет доступны 2 кнопки:

  • Повторить регистрацию. По данной кнопке автор процесса может направить задачу на повторную регистрацию. Документ при этом перейдет в состояние На регистрации. При отправке документа на повторную регистрацию будет открыто окно изменения параметров процесса, в котором автор процесса может изменить значения некоторых полей.
  • Завершить регистрацию. При нажатии этой кнопки документ останется в состоянии Не зарегистрирован и процесс регистрации завершится с отрицательным результатом.

 

Если же документ был зарегистрирован и задача на регистрацию исполнена с положительным результатом, то автору процесса в карточке задачи ознакомления с результатом регистрации будет доступна только одна кнопка: Ознакомился. При нажатии на данную кнопку процесс по регистрации документа завершается с положительным результатом.

Если задача по ознакомлению с результатом регистрации приходит автору повторно, то по ссылке История регистрации можно посмотреть предыдущие результаты выполнения регистрации документа.
