Общие сведения

Этот процесс можно создавать непосредственно из карточки, исходящего или внутреннего документа, при помощи меню Создать на основании. Также этот процесс может быть создан из задач "Ознакомиться с результатом согласования" и "Ознакомиться с результатом утверждения" при помощи меню Создать на основании. В случае отрицательного результата из этой задачи автор может отправить документ на повторную регистрацию. В случае положительного результата из этой задачи автор получает задачу на ознакомление.

При выборе исполнителя можно указать не только конкретного пользователя, но и роль, например "Секретарь".

Чем отличаются предметы процесса

Предметы процесса могут быть основными и вспомогательными.

 Основной предмет содержит важную информацию о задаче и выступает в качестве ее основания.

 Вспомогательный предмет, как правило, содержит дополнительные данные, помогающие при выполнении задач. Состояние вспомогательного предмета не меняется по ходу выполнения задач процесса. В роли вспомогательного предмета может служить любой доступный объект в программе или файл с диска.

Чтобы добавить предмет в процесс , нажмите  Добавить предмет на командной панели. Если процесс создается на основании, то его основной предмет устанавливается автоматически.

Файлы в процесс добавляются также как и предметы. Файл можно добавить либо из списка файлов программы, либо с диска. При добавлении с диска можно добавить сразу несколько файлов.

Как остановить процесс

Для остановки процесса используйте команду Остановить. При этом остановленным будет помечен сам процесс, все его невыполненные задачи, а также все вложенные и подчиненные процессы и их невыполненные задачи.

Для того чтобы сделать остановленный процесс активным используйте команду Продолжить. При этом будут сделаны активными все вложенные и подчиненные процессы и их задачи. Остановленные задачи не отображаются в списках задач (например, Мои задачи).

Как прервать процесс

Нажмите Прервать на командной панели карточки процесса.

При этом с процессом происходят следующие изменения:

  • Процесс и его задачи прерываются,
  • Подчиненные процессы и их задачи прерываются,
  • Состояние предмета "Зарегистрирован" очищается.

Прервать процесс может его автор или администратор. Прерванный процесс, в отличие от остановленного нельзя возобновить.

Как установить главную задачу

С помощью поля Главная задача в карточке можно указать, что данный процесс создан в рамках вышестоящего процесса. Это поле заполняется автоматически при создании процессов на основании задач или документов, но может быть установлено вручную.

Дерево подчиненных процессов можно посмотреть при помощи команды Задачи в панели навигации карточки процесса. Поле Главная задача и просмотр дерева подчиненных процессов доступны только в том случае, если в программе разрешено использование подчиненных процессов (см. Настройка программы).

Как посмотреть состояние процесса

Для просмотра текущего состояния процесса (списка выполненных и не выполненных задач) используйте команду Задачи в панели навигации карточки процесса.

Как процесс влияет на предметы и задачи

При старте этого процесса документу автоматически будет установлено состояние "На регистрации", а при завершении процесса положительно - состояние "Зарегистрирован". Если же процесс завершается отрицательно, то документ принимает состояние "Не зарегистрирован".

Изменения, которые производятся в карточке начатого процесса (например, редактирование содержимого полей), наследуются незавершенными задачами по процессу.

При этом в карточке процесса доступны для редактирования только те поля, которые можно изменять после старта процесса.

Как настроить автоматический старт процесса

Процессы могут начинаться автоматически после определенного события.

Например, можно сделать так, чтобы при регистрации входящего документа вида коммерческое предложение, он автоматически направлялся на рассмотрение руководителю.

Подробная инструкция по настройке автостарта доступна здесь.

Как отложить старт процесса

Старт процесса можно отложить на любое удобное время.

Например, если Вы создали несрочный процесс для загруженного сотрудника, можно указать время старта процесса - через день. К тому времени Ваш коллега справится со срочными делами.

Подробная инструкция по настройке отложенного старта процесса доступна здесь.
