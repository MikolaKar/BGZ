Разница между остановкой и прерыванием процесса

Остановленный процесс можно возобновить (команда Продолжить). Прерванный - нельзя.

При остановке процесса на "паузу" ставится сам процесс и все его вложенные и подчиненные процессы (с задачами). Остановленные задачи не отображаются в списках задач (например, Мои задачи).

При прерывании: прерывается сам процесс, все его подчиненные и вложенные процессы (с задачами), состояние предмета "Исполнен" очищается. Действие необратимо.

Как управлять запуском процесса

Для процесса можно настроить:

  • Отложенный старт - назначается на любое удобное время. Например, если вы создали несрочный процесс для загруженного сотрудника, старт можно отложить на несколько дней. Инструкция по настройке отложенного старта.
  • Автостарт - назначается после определенного события. Например: после регистрации входящего документа вида Обращение клиента (претензионное), он автоматически направлялся на исполнение в Отдел рекламаций. Инструкция по настройке автостарта.
  • Повторение - запускает процесс по указанному вами расписанию. Например, закупка бумаги для принтера в последнюю пятницу каждого месяца. Инструкция по настройке повторения.

Как контролировать процесс

Процесс можно поставить на контроль. При этом будет создана контрольная карточка с записями о том, что контролировать, кого контролировать и на какой срок.

Есть несколько способов работы с контролируемыми процессами:

  • Следить за всеми контрольными карточками в окне На контроле;
  • Контролировать сроки с помощью записей календаря;
  • Получать на электронную почту уведомления об истекающих и приближающихся контрольных сроках.

Контрольную карточку можно снять с контроля , вернуть на контроль , изменить срок контроля , пометить на удаление.

Список ваших контрольных карточек не виден остальным сотрудникам (просмотр разрешен только вам, вашим руководителям и делегатам). Это позволяет использовать любые удобные для вас комментарии к контролируемым процессам.

Чем отличаются предметы процесса

Предметы процесса могут быть основными, вспомогательными и заполняемыми.

Основной предмет содержит важную информацию о задаче и выступает в качестве ее основания.

Вспомогательный предмет содержит дополнительные данные, которые необходимы для выполнения задачи. Это может быть любой объект программы или файл с диска. По ходу выполнения задач процесса его состояние не меняется.

Заполняемый предмет требует введения каких-либо данных в процессе выполнения задачи. Задачу нельзя выполнить, пока предмет не будет заполнен.

Что такое комплексный процесс

Комплексный процесс состоит из цепочки заранее запланированных действий, которые стартуют по заранее настроенному вами порядку. Например, для вновь созданного документа запускается процесс регистрация, далее документ отправляется на согласование и, при его положительном завершении, на исполнение. 

Порядок выполнения исполнителями заданий может быть: Все сразу, По очереди или Смешанно.

Изменить порядок выполнения действий можно только в нестартованных процессах.

Посмотреть маршрут процесса и ход выполнения действий можно по командам:

  • Подзадачи - откроется список текущих и выполненных действий (процессов) и их задач;
  • Схема процесса - откроется схема комплексного процесса.

Наиболее частотные комплексные процессы вашего предприятия можно сохранить в качестве шаблонов. Данные шаблоны удобно привязать к карточке вида документа и, в дальнейшем, использовать при создании бизнес-процесса на основании этого вида документа.
