Разница между остановкой и прерыванием процесса

Остановленный процесс можно возобновить (команда Продолжить). Прерванный - нельзя.

При остановке процесса: на "паузу" ставится сам процесс и все его вложенные и подчиненные процессы (с задачами). Остановленные задачи не отображаются в списках задач.

При прерывании: прерывается сам процесс, все его подчиненные и вложенные процессы (с задачами), состояние предмета "Исполнен" очищается. Действие необратимо.

Как управлять запуском процесса

Для процесса можно настроить:

  • Отложенный старт - запуск процесса через определенный промежуток времени. Например, сразу после заключения договора создать процесс "Проверить отгрузку по договору" и отложить его запуск на месяц. Инструкция по настройке отложенного старта.
  • Автозапуск - запуск процесса после определенного события (настраивается администратором). Например, можно сделать так, чтобы зарегистрированная жалоба сразу отправлялась на рассмотрение ответственному лицу. Инструкция по настройке автозапуска.
  • Повторение - запуск процесса по указанному вами расписанию. Например, настроить запуск процесса "Закупка бумаги для принтера" на 15 число каждого месяца. Инструкция по настройке повторения.

Как контролировать процесс

Процесс можно поставить на контроль. При этом будет создана контрольная карточка с записями о том, что контролировать, кого контролировать и на какой срок.

Есть несколько способов работы с контролируемыми процессами:

  • Следить за всеми контрольными карточками в окне На контроле;
  • Контролировать сроки с помощью записей календаря;
  • Получать на электронную почту уведомления об истекающих и приближающихся контрольных сроках.

Контрольную карточку можно снять с контроля , вернуть на контроль , изменить срок контроля , пометить на удаление.

Список ваших контрольных карточек не виден остальным сотрудникам (просмотр разрешен только вам, вашим руководителям и делегатам). Это позволяет использовать любые удобные для вас комментарии к контролируемым процессам.

Чем отличаются предметы процесса

Предметы процесса могут быть основными, вспомогательными и заполняемыми.

 Основной предмет содержит важную информацию о задаче и выступает в качестве ее основания.

 Вспомогательный предмет содержит дополнительные данные, которые пригодятся при выполнении задач. Это может быть любой объект программы или файл с диска. По ходу выполнения задач процесса его состояние не меняется. 

 Заполняемый предмет требует введения каких-либо данных в процессе выполнения задачи. Задачу нельзя выполнить, пока предмет не будет заполнен.

Как добавить нескольких исполнителей

Чтобы добавить в процесс нескольких исполнителей или целое подразделение нажмите Подобрать (панель действий списка исполнителей). Одного из исполнителей можно назначить Ответственным. При этом он будет помечен значком и из своей карточки сможет просматривать задачи остальных исполнителей.

Можно установить порядок выполнения исполнителями заданий: Все сразу, По очереди или Смешанно.

Добавить нескольких исполнителей можно только в нестартованный процесс. Если процесс уже стартован, добавляйте исполнителей по одному кнопкой Добавить.

Что такое процесс Исполнение

Исполнение - это бизнес-процесс, при котором предмет или распоряжение передается на исполнение всем пользователям по списку.

Основное отличие от процесса Поручение - возможность добавить нескольких исполнителей и разделить процесс по шагам. В качестве предмета можно добавить любой объект программы: документ, файл, корреспондента, мероприятие, проект, тему и т.п.
