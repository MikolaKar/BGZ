Обработка предназначена для установления связей между отправителями в СМДО и контрагентами в информационной базе.

Порядок работы

Обработку следует запускать, когда у ответственного по работе с СМДО исполнителя появится задача "Распознать новых отправителей СМДО".

Период - период, за который будут просматриваться входящие сообщения с нераспознанными отправителями;

Файл dictionaryData.xml - файл со справочником организаций, подключенных к СМДО.

При нажатии Заполнить выполняется поиск входящих сообщений с нераспознанными отправителями, получается из файла dictionaryData.xml информация по отправителям. Заполняется верхняя табличная часть. Затем по УНП из информационной базы подбираются контрагенты для установления связи.

Для установления связи надо отметить флажками строку(и) в верхней таблице и одного контрагента в нижней таблице. Т.е. для каждой строки из верхней таблице должна быть отмечена флажком строка из нижней таблицы. После чего надо нажать кнопку "Установить соответствие".

Если в нижней таблице нет подходящего контрагента, тогда следует нажать кнопку "Создать", контрагент добавится на основании данных из файла, после чего нажать "Установить соответствие".  

Команды "Заполнить данные" и "Подобрать корреспондентов" служат для ручного обновления информации в таблицах.

"Заполнить данные" - заполняет строки верхней таблицы данными из файла. Поиск данных идет по Код СМДО.

"Подобрать корреспондентов" - заполняет нижнюю таблицу контрагентами с УНП, соответствующим УНП из верхней таблицы.

После установки соответсвия через некоторое время входящие сообщения будут загружены с правильными контрагентами.
