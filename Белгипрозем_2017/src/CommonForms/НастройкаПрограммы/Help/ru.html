Заголовок программы

В качестве заголовка рекомендуется использовать наименование вашего предприятия. Это удобно для навигации между окнами нескольких информационных баз. Изменение вступит в силу только после перезапуска активных соединений пользователей.

Использовать дополнительные реквизиты и сведения Если настройка включена, то администратор может использовать дополнительные реквизиты и сведения, например, для входящих, исходящих или внутренних документов.
Общие дополнительные значения Если флажок установлен, одни дополнительные значения можно назначать сразу для нескольких реквизитов (сведений).
Общие дополнительные сведения Если флажок установлен, то одни реквизиты (сведения) можно назначать сразу для нескольких наборов.
Использовать полнотекстовый поиск Настройка включает полнотекстовый поиск данных. Построение полнотекстового индекса будет выполняться автоматически в случае клиент-серверного варианта и вручную в случае файлового варианта. 
Использовать версионирование объектов Если включена, то программа автоматически сохраняет историю изменения реквизитов объектов.
Использовать электронные подписи Если включена, то документы можно удостоверять электронной подписью и шифровать.
Выполнять проверку ЭП на сервере Если установлена, то проверка ЭП всех объектов, кроме зашифрованных файлов, выполняется на сервере.
Настройки криптографии  Доступна, если установлена настройка Использовать электронные подписи. Выполняется настройка электронных подписей и шифрования.
Выполнять замеры производительности Включает режим замера производительности. При этом программа собирает данные о времени выполнения ключевых операций в системе. В дальнейшем с помощью обработки "Оценка производительности" можно проанализировать собранные данные.
Использовать бизнес-события Позволяет учитывать бизнес-события. Например, настроить автоматический старт процесса рассмотрения при регистрации входящего документа, или автоматически уведомлять исполнителя о поступлении новой задачи и т. п.
Срок хранения обработанных бизнес-событий Срок хранения бизнес-событий (например создание документа, регистрация документа и т.д.), в днях. Обработанные бизнес-события, срок хранения которых истек, удаляются.
Использовать уведомления Если включена, то администратор и пользователи смогут настраивать подписку на уведомления о событиях. Флажок доступен, если включена настройка Использовать бизнес-события.
Настройка уведомлений Открывает окно настроек уведомлений. Кнопка доступна, если флажок Использовать уведомления установлен.
Использовать форум Если включена, то в программе появится раздел Форум.
Использовать рабочий календарь Если включена, то в программе появится Рабочий календарь.
Использовать напоминания Если флажок установлен, то о календарных событиях можно сообщать пользователю при помощи всплывающих напоминаний.
Отображать фотографии Если включена, то в карточке пользователя, встроенной почте и в обсуждениях на форуме будут показываться фотографии пользователя. Фотографию добавляет администратор в карточке пользователя.
Использовать инструкции Включает отображение инструкций для сотрудников. Инструкции описывают какие действия и в каком порядке нужно выполнить, чтобы исполнить поручение, поставить задачу и т. д.
Загрузить типовые инструкции Программа уже содержит ряд готовых инструкций, которые можно использовать как есть, или предварительно отредактировать.
Использовать контроль объектов Включает механизм контроля объектов, который позволяет поставить на контроль объекты системы, контролировать срок и ход выполнения работ по ним.
Настроить доступ в Интернет на сервере 1С:Предприятия Нажатие кнопки открывает форму настройки прокси сервера.

Права доступа

Настройка

Описание

Использовать ограничение прав доступа

Включает механизм ограничения прав доступа на файлы и документы.

Если флажок не установлен, то все пользователи получают неограниченный доступ к данным программы.

Не рекомендуется использовать ограничение прав доступа в файловом варианте информационной базы, т.к. это может привести к существенному замедлению работы программы.

Отложенное обновление прав доступа

Если флажок установлен, то обновление прав выполняется в фоновом режиме, не мешая работе пользователей.

Рекомендуется включить эту настройку, чтобы не замедлять работу программы.

Добавлять руководителям доступ подчиненных

Если включена, то доступ подчиненных автоматически добавляется к доступу руководителей.

Использовать группы пользователей

Позволяет настроить доступ для групп пользователей. Затем ограничивать права просто добавляя пользователей в эти группы.

Запрещать вход в программу без пароля

Если флажок установлен, то при входе в программу у пользователей будет запрошен пароль.

Проверять сложность пароля

Если включена, то пароль будет проверяться на сложность при попытке его изменения.

Ограничивать доступ через веб-серверы

Если включена, то пользователи смогут подключиться к программе только через разрешенные веб-серверы, которые можно указать в карточке пользователя.

Использовать учет доступа к персональным данным Если флажок установлен, то в карточках файлов, версий файлов, входящих, исходящих и внутренних документов станет доступен список физических лиц, персональные данные которых в них содержатся.
Также при установке флажка в журнал регистрации будут автоматически записываться сведения о доступе к документам, файлам и версиям файлов. 
Протоколировать работу пользователей Если включена, то для контроля и анализа работы пользователей программа протоколирует основные операции: добавление, изменение, получение информации (открытие карточки, поиск).
Срок хранения протоколов Укажите срок хранения (в месяцах) записей протокола работы пользователей. Записи, срок хранения которых истек, удаляются автоматически. Если указан 0, то записи хранятся бессрочно и не удаляются автоматически.
Каталог хранения протоколов

Сетевой каталог для временных файлов. 
Каталог должен быть доступен на чтение и запись со всех серверных компьютеров.
Если в кластере 1С:Предприятия только один сервер, то задавать каталог для временных файлов не требуется.
Если сервера в кластере установлены на компьютерах под управлением разных операционных систем, нужно задать два каталога: для Windows и для Linux, которые физически указывают на один и тот же сетевой каталог.
Записи сохраняются в архивированном файле формата MXL.

Делопроизводство

Настройка Описание
Использовать номенклатуру дел Если настройка установлена, то в программе доступны справочники Номенклатура дел, Дела (тома) и реквизит Помещен в дело в карточках входящих, исходящих и внутренних документов.
Использовать журнал передачи Если включена, то пользователи могут вести и просматривать журнал передачи бумажных документов исполнителям.
Использовать состав документов Если включена, то в карточках входящих, исходящих и внутренних документов отображаются поля для указания количества листов, приложений и листов в приложениях.
Использовать виды входящих документов Если включена, то в карточке входящего документа появляется обязательное для заполнения поле Вид.
Использовать виды исходящих документов Если флажок установлен, то в карточке исходящего документа появляется обязательное для заполнения поле Вид.
Использовать виды внутренних документов Если флажок установлен, то в карточке внутреннего документа появляется обязательное для заполнения поле Вид.
Использовать файлы у входящих документов Если флажок установлен, то в карточке входящего документа появляется возможность присоединить файлы и работать с ними.
Использовать файлы у исходящих документов Если установлена, то в карточке исходящего документа появляется возможность присоединить файлы и работать с ними.
Использовать связи документов Если флажок установлен, то в карточках всех документов появляется возможность настройки перекрестных ссылок документов друг на друга.
Использовать резолюции в документах Если настройка установлена, то в карточке входящих документов появляется поле для ввода резолюции руководителя по документу.
Использовать списки рассылки по корреспондентам Если настройка установлена, то при оформлении исходящего документа можно указать список рассылки в качестве получателя документа.
Использовать визы согласования Если настройка установлена, то в карточке исходящего и внутреннего документа отображается таблица для работы с визами согласования документа.
Использовать учет по организациям

Если настройка установлена, то программа будет вести учет всех ключевых данных в разрезе нескольких организаций. Для этого в карточках документов, номенклатуры дел и шаблонов процессов появится поле Организация.

Если настройка не установлена, то в данной форме отображается закладка Данные предприятия, в которой следует указать информацию о предприятии.

Использовать вопросы деятельности Если настройка установлена, то в разделе Нормативно-справочная информация появится команда Вопросы деятельности, в документах появится поле Вопрос деятельности. Также можно будет включать индекс вопроса деятельности в формат регистрационного номера.
Использовать учет обращений граждан

Настройка включает учет обращений граждан: в настройках вида входящего документа появляется признак Является обращением от граждан, список обращений граждан отображается в разделе Документы и файлы, доступны отчеты по обращениям граждан.

Использовать управление мероприятиями Настройка позволяет организовывать, учитывать и анализировать совещания, собрания, обсуждения, заседания, корпоративные мероприятия и т.п.
Основная валюта

Основная валюта используется при создании документов с включенным учетом суммы. В основную валюту пересчитываются суммы сводных отчетов, сформирванные в разных валютах.

Использовать грифы доступа Если установлена, то доступ к входящим, исходящим и внутренним документам будет ограничен грифами доступа.
Использовать состояния документов Позволяет отключить весь механизм состояний документов, при этом поле Состояние не будет отображаться в карточках документов, в списках, отчетах и т.п.
Разрешить изменение состояние вручную Настройка позволяет пользователям вручную менять состояния входящих, исходящих и внутренних документов. Если настройка не установлена, то состояния документов будут изменяться только процессами автоматически.
Ограничивать доступность полей по состоянию Позволяет настроить доступность полей карточек и операций с документами в зависимости от состояния документа в справочнике Настройки доступности по состоянию.

Категории для документов и файлов

 Если настройка включена, то документам и файлам можно назначать категории.
Автоматическая категоризация документов и файлов Настройка определяет, использовать или нет правила автоматического назначения категорий документам и файлам.
Проверка категорий документов и файлов Если настройка установлена, то становится доступным инструмент для проверки правильности назначения категорий документам и файлам.
Использовать штрихкодирование документов и файлов

Если установлена, то становятся доступными функции, связанные со штрихкодированием документов и файлов

Файлы

Настройка Описание
Использовать хранилище файлов Если флажок установлен, то  можно хранить и обрабатывать файлы, организуя их в структуру папок.
Максимальный размер файла Программа запрещает добавлять в хранилище файлы, размер которых превышает указанный. По умолчанию максимальный размер файла равен 50 Мб.
Хранить файлы в томах на диске Если флажок установлен, то файлы, прикрепленные к карточкам документов, будут сохраняться не в информационной базе, а в файловом хранилице на выделенном для этих целей сервере.
Извлекать тексты только на сервере

Указывает, что серверу необходимо самостоятельно извлекать тексты из файлов, для построения полнотекстового индекса. При этом извлечение текста выполняется регламентным заданием автоматически.
Извлечение текстов работает только на серверах под управлением ОС Windows.

Список расширений файлов OpenDocument Строка, содержащая список расширений файлов формата OpenDocument (например OpenOffice). По умолчанию "ODT OTT ODP OTP ODS OTS ODC OTC ODF OTF ODM OTH SDW STW SXW STC SXC SDC SDD STI". Из файлов формата OpenDocument текст извлекается внутри конфигурации.
Использовать автозаполнение шаблонов файлов

Настройка включает автозаполнение файла. То есть файл, созданный по шаблону, автоматически запоняется данными документа, к которому он прикреплен.

Запретить загрузку файлов с расширением С помощью этого флажка можно запретить загружать в информационную базу файлы с указанными расширениями. Список запрещенных расширений по умолчанию - "COM, EXE, BAT, CMD, VBS, VBE, JS, JSE, WSF, WSH, SCR"). При необходимости можно добавить другие расширения.
Настройка автоматической загрузки файлов Открывает окно настройки автоматической загрузки файлов, которая выполняется с помощью регламентного задания.
Использовать распознавание изображений с помощью CuneiForm

Если флажок установлен, то при загрузке файла с расширением bmp tif tiff jpg jpeg png gif программа автоматически распознает его и преобразует в текстовый формат. Для работы распознавания должна быть установлена программа CuneiForm v12 (http://www.cuneiform.ru/downloads/index.html).
Внимание: Установку программы CuneiForm следует делать под тем пользователем, который в дальнейшем будет с ней работать. 
Для распознавания файлов в формате PDF

Особенности установки CuneiForm для Windows Server 2008
Для установки следует выполнить следующие действия:

  • Установите CuneiForm на другой компьютер (например, с ОС Windows XP);
  • Скопируйте каталог вида "C:\Program Files (x86)\Cognitive\CuneiForm\" на компьютер с Windows Server 2008;
  • В каталоге CuneiForm на компьютере с Windows Server 2008 запустите Command Prompt с правами администратора (в меню "Пуск-Все программы-Стандартные-Command Prompt" в контекстном меню следует выбрать Run as...).
  • В Command Prompt выполните команду regsvr32 APuma.dll.
Автоматически очищать устаревшие версии файлов Если настройка включена, версии с истекшим сроком хранения (по умолчанию 180 дней) будут сохраняться в указанный каталог и очищаться в базе. Это касается только неактивных версий.
Срок устаревания неактивных версий

Неактивные версии хранятся, если к ним не было обращений, указанное число дней.

Путь сохранения удаляемых версий Путь в UNC формате (\\server\path), куда сохраняются удаляемые из базы или тома версии.
Ответственный за удаление неактивных версий Пользователь, получающий уведомления об удалении версий с истекшим сроком хранения.
Тип оповещения ответственного Настройка указывает, как оповещать ответственного об удалении версий с истекшим сроком хранения - по электронной почте или с помощью задачи процесса Ознакомление.
Папка для хранения отчетов об удалении Если указан тип оповещения Задача, то файлы отчетов складываются в данную папку и затем указываются как предмет процесса.

Процессы и задачи

Настройка Описание
Использовать процессы и задачи Если флажок установлен, то пользователям становится доступен раздел Совместная работа и разрешена работа с задачами и процессами.
Разрешить изменение даты исполнения задач Настройка разрешает пользователям менять дату исполнения в карточках задач.
Использовать перенаправление задач Настройка разрешает исполнителям перенаправлять свои задачи другим исполнителям.
Использовать дату и время в сроках задач Если флажок установлен, то в задачах и процессах даты будут отображаться с точностью до минут.
Использовать внешние задачи Настройка разрешает использовать внешние роли в адресации процессов. Эта настройка доступна только если установлен флажок Использовать обмен данными.
Настроить расписание уведомления исполнителей по электронной почте Выполняется настройка расписания регламентного задания для уведомления исполнителей.
Использовать фоновую маршрутизацию комплексных процессов Если флажок установлен, то маршрутизация комплексных процессов выполняется в фоновом режиме, не мешая работе пользователей. По умолчанию настройка включена. Возможность фоновой маршрутизации доступна только для комплексных процессов в клиент-серверной базе.
Использовать повторение процессов

Если флажок установлен, то в карточках и списках процессов появится команда Настроить повторение, которая позволит указать расписание (начало повторения, окончание повторение, повторять ежедневно, еженедельно, ежемесячно). Процесс будет автоматически повторяться в соответствии с указанным расписанием. 

Использовать рассмотрение вопросов выполнения задач

Настройка позволяет задать вопрос коллеге из карточки любой задачи.  

Вести учет переносов сроков задач Если флажок установлен, пользователь может отправить автору задачи запрос на перенос срока ее исполнения.  
Использовать графики работы Если настройка включена, то срок задач процессов будет устанавливаться с учетом рабочего времени сотрудника. Общий график работы сотрудников указывается в настройках системы, особенный график работы некоторого подразделения или некоторого сотрудника указывается в карточке подразделения (пользователя).
Максимальный общий размер файлов в уведомлении Ограничивает максимальный объем файлов, передаваемых в одном уведомлении.
Срок хранения протоколов выполнения задач по почте

Ограничивает количество дней для хранения записи в Протоколе исполнения задач по почте после завершения задачи.

Проекты и трудозатраты

Настройка Описание
Вести учет по проектам

Настройка позволяет вести учет в разрезе проектов: автоматизировать операционную деятельность по исполнению плана проекта и проводить аналитический учет данных по проектам.

Использовать виды проектов

Настройка позволяет разделять проекты по видам. 

Вести учет плановых трудозатрат в бизнес-процессах

Если флажок установлен, то во всех бизнес-процессах появляются реквизиты, в которых можно указать плановые трудозатраты для исполнителей.

Вести учет фактических трудозатрат Если флажок установлен, то после завершения работы с объектом (задачей, письмом, документом и т.п.) программа предложит пользователю ввести по нему трудозатраты.
Использовать ежедневные отчеты

При установленном флажке становится доступен Ежедневный отчет, в котором можно указать к какому проекту относятся трудозатраты.

Отчет автоматически заполняется трудозатратами, указанными при помощи хронометража или команды Указать трудозатраты.

Основная единица длительности

Указывается основная единица длительности исполнения задач по проектам. По умолчанию - час.

Основная единица трудозатрат Указывается основная единица трудозатрат по проектам. ПО умолчанию - человеко-час.

Почта

Настройка Описание
Использовать встроенную почту

Если натройка включена. то учет входящей и исходящей электронной почты ведется непосредственно в программе.

Префикс для ответных писем

Необходимо указать префикс, который будет автоматически добавляться в начало темы ответного пписьма. По умолчанию "Re:".

Префикс для пересылаемых писем

Необходимо указать префикс, который будет автоматически добавляться в начало темы пепресылаемого письма. По умолчанию "Fw:".

Сворачивать множественные префиксы Настройка позволяет сворачивать префиксы при ответе и пересылке. Например, вместо "Re:Re:Re:Re:" в теме письма будет добавлено "Re[4]:"
Вести протокол доставки почты

Включает протоколирование операций получения и отправки писем. Протокол доступен для полноправных пользователей в панели навигации карточки письма или карточки учетной записи.

Срок хранения протокола доставки почты

Необходимо указать период времени, в днях, за который должны храниться записи протокола доставки почты.

Использовать внутреннюю маршрутизацию

Если флажок установлен, то доставка писем, отправленных внутренним адресатам, будет проходить не через сервер электронной почты, а внутри "1С:Документооборота".

Внимание: Применение внутренней маршрутизации значительно ускорит доставку внутренней электронной почты.

Выполнять отправку в отдельном регламентном задании

Если настройка включена, то отправка почты выполняется отдельным регламентным заданием ОтправкаПисем, что значительно ускорит отправку.

Очистка устаревших HTML представлений содержания писем

HTML представления содержания писем хранятся в программе. Для регулярной очистки укажите необходимость автоматической очистки устаревших HTML представлений содержания писем и срок устаревания.
Чтобы очистить все устаревшие HTML представления содержания писем нажмите кнопку Очистить сейчас.

Вариант шапки письма при ответе Выберите полный (в 4 строки) или  сокращенный (в 1 строку) вариант шапки исходного письма при ответе.
Вариант шапки письма при пересылке Выберите полный (в 4 строки) или сокращенный (в 1 строку) вариант шапки исходного письма при пересылке.
Сокращать список адресатов при большом числе адресатов Если установлен, то при большом числе адресатов список адресатов будет показан сокращенно. 
Число адресатов для сокращения списка Укажите число  адресатов для сокращения списка.
Помечать строки исходного письма при ответе Если установлен, то строки исходного письма при ответе будут помечаться.
Помечать строки исходного письма при пересылке Если установлен, то строки исходного письма при пересылке будут помечаться.
Цитирование в переписке: текстовый формат Выберите способ цитирования в переписке для писем в текстовом формате. Значение по умолчанию ">".
Цитирование в переписке: формат HTML Выберите способ цитирования в переписке для писем в формате HTML - вертикальная полоса или отступ.
Максимальный размер внешнего письма Программа запрещает отправку внешних писем, размер которых превышает указанное ограничение. Значение по умолчанию: 20 МБ. Отмена ограничения: 0.

Настройка Описание

Префикс узла распределенной информационной базы Указывается префикс узла текущей информационной базы. Доступен, если флажок Использовать обмен данными установлен.

Является сервером СВД Если установлена, то данная информационная база может быть использована в качестве сервера СВД.
