В данном окне можно настроить подписки и указать адреса для уведомлений пользователей. 

Настройка уведомлений администратором

На закладке Уведомления по умолчанию можно указать, какие уведомления пользователи системы должны получать по умолчанию.

Настройка Разрешить пользователям изменять настройки уведомлений определяет, смогут ли пользователи самостоятельно изменять состав получаемых уведомлений.

  • Если настройка установлена – пользователи смогут подписаться или отписаться от любых уведомлений, в том числе от уведомлений, установленных по умолчанию.
  • Если отключена - пользователи всегда будут получать уведомления по умолчанию без возможности отказаться от них. На остальные уведомления пользователи могут подписаться или отписаться по желанию.

На закладке Адреса для уведомлений можно посмотреть, какие адреса для уведомлений указаны у пользователей. Если адрес не указан, строка выделена красным цветом и находится в начале списка. Необходимо выбрать данную строку и указать почтовый адрес в открывшейся карточке пользователя.

По умолчанию для уведомлений используются адреса почты, указанные в контактной информации пользователя. Пользователь не сможет отказаться от получения этих уведомлений. Дополнительные адреса рассылки пользователь может указать в персональных настройках пользователя.

Рассылка уведомлений

Для рассылки уведомлений необходимо выполнить два регламентных задания:

  1. Сначала Обработка произошедших бизнес-событий. При этом будут готовы уведомления, созданные на основании произошедших бизнес-событий, и сформируется очередь рассылки уведомлений.
  2. После этого Уведомление пользователей о произошедших событиях - для непосредственной рассылки полготовленных уведомлений.

Контроль подхода срока задач

Если установлена функциональная опция Использовать графики работы, тогда срок задачи считается с учетом графика работы конкретного пользователя. В противном случае, считаются просто дни до выполнения задачи, без учета того, является ли день будним, выходным или праздником.

Для контроля подхода срока задач:

  1. Сначала выполните регламентное задание Контроль приближения сроков задач.
  2. После этого выполните регламентное задание Обработка произошедших бизнес-событий. Произойдет постановка уведомлений, сформированных на основании произошедших бизнес-событий, и формирование очереди рассылки уведомлений.
  3. И после этого выполните регламентное задание Уведомление пользователей о произошедших событиях для непосредственной рассылки сформированных уведомлений.

Виды подписок на уведомления

Процессы и задачи:

  • Ход выполнения процесса - пользователь будет получать уведомления о выполнении задач тех процессов, автором которых он является;
  • Остановка и прерывание процессов - пользователь будет получать уведомления о том, что был прерван другим пользователем бизнес-процесс, автором которого он является или прерван бизнес-процесс, исполнителем задачи которого он является.
  • Поступление новой задачи - пользователь будет получать уведомления о новых задач, исполнителем которых он является;
    • Использовать выполнение задач по почте - если флажок установлен, то исполнители могут выполнять задачи с помощью обычного почтового клиента (в т. ч. веб-почты), не входя в "1С:Документооборот", например, в командировке.
    • Кроме этого, должно быть включено регламентное задание Исполнение задач по почте, а у исполнителей в настройках уведомлений установлен флажок Исполнение задач по почте, и указан почтовый адрес для получения уведомлений;
    • Когда поступает задача, от программы приходит письмо с описанием задачи и вариантами ее выполнения в виде гиперссылок.
    • Для выполнения задачи достаточно щелкнуть на нужный вариант. При этом будет сформировано готовое ответное письмо с результатом выполнения. Пись мо можно дополнить комментарием или прикрепить файлы (если требуется).
    • Если ответное сообщение не было распознано, или произошли какие-то ошибки, то исполнителю вернется уведомление о необходимости повторного выполнения задачи.
    • Все события в ходе выполнения задач протоколируются. Протокол находится в разделе Настройка и администрирование по команде Протокол выполнения задач по почте.
    • Кроме того, результаты выполнения задач заносятся в историю выполнения задач.
  • Поступление перенаправленной задачи - пользователь будет получать уведомления о перенаправлении ему задач;
  • Отмена выполнения задач – пользователь будет получать уведомления о том, что выполненные им задачи были отменены другим пользователем;  
  • Приближение сроков выполнения задач - пользователь будет получать уведомления о том, что подошел срок выполнения задач, исполнителем которых он является;
  • Контроль просроченных задач - пользователь будет получать уведомления о том, что задачи, исполнителем которых он является, просрочены;

Контроль

  • Приближение контрольного срока - контролер получит уведомление о приближении контрольного срока за указанное количесто дней.
  • Уведомление о просроченном контроле - предупредит о том, что есть объекты, срок контроля которых истек.

Документы

  • Приближение окончания срока действия документов - пользователь будет получать уведомления о том, что скоро окончится срок действия документов, ответственным за которые он является;
  • Окончание срока действия документов - пользователь будет получать уведомления о том, что закончился срок действия документов, ответственным за которые он является;

Прочее

  • Появление ответа в форуме - пользователь будет получать уведомления о том, что появился новый ответ на его сообщение, на его тему или на тему предмета, с которой он связан (например, ответ в тему обсуждения внутреннего документа, ответственным за который он является);
  • Уведомления программы - программные уведомления об ошибках и важных событиях, которые имеют прямое отношение к пользователю. Например, если пользователь настроил отложенный старт процесса, а при старте произошла ошибка - программа пришлет сообщение об этом.

Настройка системной учетной записи почты

Для работы механизма уведомлений необходимо, чтобы администратор системы настроил системную учетную запись почты. Именно данная запись будет использована для рассылки уведомлений. Необходимо так же убедится, что у неё установлена галочка Использовать: Для "легкой" почты и её можно использовать для отправки писем.

Включение уведомлений администратором

Для включения функциональной опции использования уведомлений необходимо перейти в Настройка и администрирование – Сервис – Настройка программы и на закладке Общие параметры включить Использовать бизнес-события и Использовать уведомления.

Настройка уведомлений пользователем

Для настройки уведомлений пользователь может перейти Настройка и администрирование – Сервис - Персональные настройки - Настройка уведомлений….

На закладке Подписки пользователь может подписаться на различные уведомления, которые он хочет. Например, для получения уведомлений о приближении сроков выполнения задач необходимо на закладке Подписки установить флаг Приближение сроков выполнения задач, и указать срок, за который следует уведомлять пользователя (по умолчанию – 3 дня).

Для подписки на другие события необходимо, например, из карточки внутреннего документа с помощью команды Подписаться… выбрать события, по которым пользователь хочет получать уведомления. Такие подписки отображаются в персональных настройках на закладке Подписки на изменение данных.

После этого пользователь должен указать способ доставки уведомлений на закладке Способы доставки. Здесь он может указать один или несколько адресов, на которые ему будут приходить уведомления. Если не добавить хотя бы один e-mail адрес в Способы доставки то пользователь уведомления будет получать только на адрес, указанный в карточке пользователя.
